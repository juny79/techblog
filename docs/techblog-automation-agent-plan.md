## 1. 개요

본 보고서는 티스토리(Tistory) 플랫폼에 기술블로그 콘텐츠를 **자동으로 생성·편집·게시**하기 위한 다중 에이전트(Multi-Agent) 시스템의 구성 계획을 기술한다.

각 에이전트는 독립적으로 특정 역할을 담당하며, 오케스트레이터(Orchestrator)가 이들을 조율하여 **주제 선정 → 콘텐츠 작성 → 품질 검수 → 자동 게시**에 이르는 전체 파이프라인을 완전 자동화하는 것을 목표로 한다.

---

## 2. 목표 및 범위

### 2.1 핵심 목표

| 목표 | 설명 |
|------|------|
| **콘텐츠 자동 생성** | 트렌드 기반 기술 주제를 자동 탐색하고 고품질 블로그 포스트 작성 |
| **일관된 퀄리티 유지** | 품질 검수 에이전트를 통해 게시 전 자동 검토 |
| **SEO 최적화** | 검색 노출을 위한 키워드 및 메타데이터 자동 최적화 |
| **완전 자동 게시** | 티스토리 API를 통한 무인 게시 자동화 |
| **성과 분석 및 개선** | 게시 후 통계 수집 및 다음 콘텐츠에 피드백 반영 |

### 2.2 범위

- **대상 플랫폼:** 티스토리(Tistory) 블로그
- **콘텐츠 유형:** 기술 튜토리얼, 개념 설명, 코드 예제, 기술 트렌드 분석
- **자동화 수준:** 완전 자동(Fully Automated) 또는 사람 검토 후 게시(Human-in-the-Loop) 선택 가능

## 4. 기능별 에이전트 구성

### 4.1 주제 탐색 에이전트 (Topic Discovery Agent)

#### 역할
- 기술 트렌드와 인기 주제를 자동으로 탐색 및 선정

#### 입력 소스
- GitHub Trending 저장소
- Hacker News, Dev.to, Medium Tech
- Google Trends API
- Stack Overflow 인기 질문
- 기존 블로그 포스트 성과 데이터 (피드백 에이전트로부터)

#### 처리 로직
1. 각 소스에서 최신 기술 키워드 및 주제 수집
2. 중복 제거 및 카테고리 분류
3. 트렌드 점수 및 경쟁 강도 산출
4. 미게시 주제 우선순위 목록 생성

#### 출력
```json
{
  "topic_id": "uuid",
  "title": "2026년 주목해야 할 Rust 웹 프레임워크 Top 5",
  "keywords": ["Rust", "웹 프레임워크", "Actix", "Axum"],
  "category": "백엔드",
  "trend_score": 92,
  "estimated_search_volume": 1200,
  "priority": "high"
}
```

#### 기술 구현
- **언어:** Python
- **라이브러리:** `feedparser`, `requests`, `BeautifulSoup4`, `pytrends`
- **스케줄:** 매일 오전 6시 실행

---

### 4.2 콘텐츠 생성 에이전트 (Content Generation Agent)

#### 역할
- 선정된 주제를 바탕으로 블로그 포스트 본문을 자동 작성

#### 입력
- 주제 탐색 에이전트의 출력 (주제, 키워드, 카테고리)
- 블로그 톤앤매너 가이드라인 (설정 파일)
- 참고 자료 URL 목록 (선택)

#### 처리 로직
1. LLM(대형 언어 모델)에 구조화된 프롬프트 전달
2. 서론 → 본론(섹션별) → 결론 구조로 글 생성
3. 마크다운 형식으로 출력
4. 생성된 내용에서 코드 블록 섹션 추출 → 코드 예제 에이전트로 전달

#### 프롬프트 템플릿 (예시)
```
역할: 당신은 한국어 기술 블로그 전문 작가입니다.
주제: {topic_title}
키워드: {keywords}
독자 수준: {audience_level}
분량 목표: {word_count}자 이상
출력 형식: 마크다운
요구사항:
  - 실용적인 예제 포함
  - 각 섹션에 명확한 소제목
  - 결론에 핵심 요약 포함
```

#### 출력
- 마크다운 형식의 블로그 본문 (2,000~4,000자)
- 코드 블록 목록 (코드 에이전트 연동용)

#### 기술 구현
- **LLM:** OpenAI GPT-4o 또는 Anthropic Claude 3.7
- **SDK:** `openai`, `anthropic`
- **재시도 로직:** 품질 기준 미달 시 최대 3회 재생성

---

### 4.3 코드 예제 생성 에이전트 (Code Example Agent)

#### 역할
- 블로그 본문 내 코드 예제의 정확성 검증 및 실행 가능한 샘플 코드 생성

#### 입력
- 콘텐츠 에이전트에서 추출된 코드 블록
- 프로그래밍 언어 정보

#### 처리 로직
1. 코드 문법 분석 및 린팅 (예: `pylint`, `eslint`)
2. 샌드박스 환경에서 코드 실행 및 출력 검증
3. 오류 발생 시 LLM을 통한 자동 수정
4. 최종 검증된 코드로 본문 업데이트

#### 지원 언어
- Python, JavaScript/TypeScript, Java, Go, Rust, Kotlin, Swift

#### 기술 구현
- **샌드박스:** Docker 컨테이너 기반 격리 실행 환경
- **언어별 실행기:** `subprocess` + 언어별 런타임
- **타임아웃:** 코드 실행 당 최대 30초

### 4.4 이미지 생성 에이전트 (Image Generation Agent)

#### 역할
- 블로그 썸네일 및 본문 삽입 이미지 자동 생성

#### 입력
- 포스트 제목 및 핵심 키워드
- 카테고리별 이미지 스타일 설정

#### 처리 로직
1. 주제 기반 이미지 생성 프롬프트 자동 작성
2. AI 이미지 생성 API 호출
3. 브랜드 가이드라인에 맞는 텍스트 오버레이 추가
4. 티스토리 최적 해상도(1200x628px)로 리사이징

#### 이미지 유형
| 유형 | 용도 | 해상도 |
|------|------|--------|
| 썸네일 | 포스트 대표 이미지 | 1200x628px |
| 섹션 이미지 | 본문 내 시각적 설명 | 800x450px |
| 다이어그램 | 아키텍처/플로우 도식화 | 자동 조절 |

#### 기술 구현
- **이미지 생성 AI:** DALL-E 3, Stable Diffusion, Midjourney API
- **후처리:** `Pillow`, `cairosvg` (다이어그램 SVG → PNG 변환)
- **다이어그램:** Mermaid.js 자동 렌더링

### 4.5 SEO 최적화 에이전트 (SEO Optimization Agent)

#### 역할
- 게시 전 검색 최적화를 위한 메타데이터 및 본문 보강

#### 입력
- 완성된 블로그 본문 (마크다운)
- 타겟 키워드 목록

#### 처리 로직
1. **키워드 밀도 분석:** 타겟 키워드 적절 분포 확인 (1~3%)
2. **제목 최적화:** 클릭율(CTR) 높은 제목 형식으로 수정 제안
3. **메타 설명 생성:** 160자 이내 요약 문장 자동 작성
4. **내부/외부 링크 삽입:** 관련 포스트 자동 링크 및 공신력 있는 외부 참조 추가
5. **태그 및 카테고리 자동 지정:** 티스토리 태그 시스템 연동
6. **가독성 점수 산출:** Flesch-Kincaid 기반 한국어 가독성 평가

#### 출력
```json
{
  "seo_title": "최적화된 포스트 제목",
  "meta_description": "160자 이내 검색 결과 설명문",
  "tags": ["Rust", "웹개발", "백엔드", "2026"],
  "category": "백엔드",
  "readability_score": 78,
  "keyword_density": {
    "Rust": 2.1,
    "웹 프레임워크": 1.8
  }
}
```

#### 기술 구현
- **SEO 분석:** `beautifulsoup4`, 자체 키워드 분석 모듈
- **가독성:** `konlpy` (한국어 형태소 분석)

### 4.6 품질 검수 에이전트 (Quality Review Agent)

#### 역할
- 게시 전 최종 품질 게이트 역할 수행

#### 검수 항목

| 검수 항목 | 판단 기준 | 통과 조건 |
|----------|----------|---------|
| 글자수 | 최소 1,500자 이상 | ✅ 통과 / ❌ 재생성 |
| 코드 실행 | 모든 코드 블록 오류 없음 | ✅ 통과 / ❌ 코드 에이전트 재처리 |
| 표절 검사 | 유사도 20% 이하 | ✅ 통과 / ❌ 재작성 |
| SEO 점수 | 70점 이상 | ✅ 통과 / ❌ SEO 에이전트 재처리 |
| 이미지 첨부 | 썸네일 필수, 본문 이미지 1개 이상 | ✅ 통과 / ❌ 이미지 에이전트 재실행 |
| 오타/맞춤법 | 맞춤법 오류 0개 | ✅ 통과 / ❌ 자동 교정 |

#### 처리 로직
1. 각 검수 항목 순차 검토
2. 실패 항목 발생 시 해당 에이전트에 재처리 요청
3. 최대 3회 재처리 후 여전히 실패 시 **사람 검토 큐(Human Review Queue)**로 이관
4. 모든 항목 통과 시 게시 에이전트로 전달

#### 기술 구현
- **맞춤법 검사:** `py-hanspell` (Naver 맞춤법 검사기 연동)
- **표절 검사:** 외부 API (CopyLeaks, Copyleaks Korean 지원) 또는 내부 유사도 비교

### 4.7 게시 에이전트 (Publishing Agent)

#### 역할
- 검수 완료된 콘텐츠를 티스토리에 자동 게시

#### 입력
- 품질 검수 통과된 포스트 전체 데이터

#### 티스토리 API 연동 절차
1. **OAuth 2.0 인증** 토큰 갱신 (필요 시)
2. 이미지 파일 티스토리 첨부 파일 API로 업로드 → URL 획득
3. 마크다운 → 티스토리 HTML 변환
4. 포스트 생성 API 호출 (제목, 본문, 카테고리, 태그, 공개 여부 포함)
5. 게시 성공 시 URL 및 포스트 ID 저장

#### 게시 옵션 설정
```yaml
publishing:
  visibility: public          # public / private / protected
  category_id: 12             # 티스토리 카테고리 ID
  scheduled_time: null        # null이면 즉시 게시, 또는 ISO 8601 형식
  notify_subscribers: true    # 구독자 알림 여부
  comment_enabled: true       # 댓글 허용 여부
```

#### 기술 구현
- **API:** 티스토리 Open API (OAuth 2.0)
- **HTML 변환:** `markdown2`, `mistune`
- **에러 처리:** 게시 실패 시 최대 3회 재시도 후 관리자 알림

---

### 4.8 분석 및 피드백 에이전트 (Analytics & Feedback Agent)

#### 역할
- 게시된 포스트 성과를 수집하고 시스템 개선에 피드백

#### 수집 데이터
- 포스트별 조회수, 좋아요, 댓글 수
- 검색 유입 키워드 (Google Search Console API)
- 체류 시간, 이탈률 (Google Analytics 4 API)
- 소셜 공유 수 (선택)

#### 피드백 루프
1. 매주 성과 데이터 수집 및 분석
2. 높은 성과 포스트의 특성 추출 (주제 유형, 길이, 키워드 등)
3. 주제 탐색 에이전트 및 콘텐츠 생성 에이전트의 설정값 자동 조정
4. 월간 성과 리포트 자동 생성 및 이메일 발송

#### 기술 구현
- **Analytics:** Google Analytics 4 Data API
- **Search Console:** Google Search Console API
- **리포트:** `pandas`, `matplotlib` 기반 자동 리포트 생성

## 5. 에이전트 간 통신 및 오케스트레이션

### 5.1 오케스트레이터 역할
- 전체 파이프라인 상태 관리 (State Machine)
- 에이전트 실행 순서 제어
- 실패/재시도 로직 총괄
- Human-in-the-Loop 큐 관리

### 5.2 통신 방식

```
[메시지 큐 기반 비동기 통신]

오케스트레이터
    │
    ├─ Redis Pub/Sub 또는 RabbitMQ
    │
    ├─ 각 에이전트는 독립 Worker 프로세스로 동작
    │
    └─ 작업 결과는 공유 데이터베이스(PostgreSQL)에 저장
```

### 5.3 상태 머신 정의

```
IDLE → TOPIC_DISCOVERY → CONTENT_GENERATION → CODE_REVIEW
     → IMAGE_GENERATION → SEO_OPTIMIZATION → QUALITY_CHECK
     → [HUMAN_REVIEW (선택)] → PUBLISHING → ANALYTICS → IDLE
```

### 5.4 재시도 정책

| 에이전트 | 최대 재시도 | 재시도 간격 |
|---------|-----------|-----------|
| 주제 탐색 | 3회 | 지수 백오프 |
| 콘텐츠 생성 | 3회 | 즉시 |
| 코드 예제 | 5회 | 즉시 |
| 이미지 생성 | 3회 | 5초 간격 |
| SEO 최적화 | 2회 | 즉시 |
| 품질 검수 | 3회 (항목별) | 즉시 |
| 게시 | 3회 | 지수 백오프 |

## 8. 티스토리 API 연동 방안

### 8.1 티스토리 Open API 인증

티스토리는 **OAuth 2.0** 방식의 인증을 제공한다.

```
1. 티스토리 개발자 앱 등록
   - https://www.tistory.com/guide/api/manage/register
   - Client ID, Client Secret 발급

2. Authorization Code 방식 인증 흐름:
   사용자 → 티스토리 로그인 → Authorization Code 획득
   → Access Token 교환 → Refresh Token 저장

3. 토큰 갱신: Access Token 만료 시 Refresh Token으로 자동 갱신
```

### 8.2 주요 API 엔드포인트

| API | 엔드포인트 | 용도 |
|-----|-----------|------|
| 블로그 정보 | `GET /v1/blog/info` | 블로그 카테고리 조회 |
| 글 작성 | `POST /v1/post/write` | 포스트 게시 |
| 글 수정 | `POST /v1/post/modify` | 포스트 수정 |
| 파일 첨부 | `POST /v1/post/attach` | 이미지 업로드 |
| 글 목록 | `GET /v1/post/list` | 기존 포스트 목록 |

### 8.3 게시 요청 예시

```python
import httpx

async def publish_post(access_token: str, post_data: dict) -> dict:
    url = "https://www.tistory.com/apis/post/write"
    params = {
        "access_token": access_token,
        "output": "json",
        "blogName": post_data["blog_name"],
        "title": post_data["title"],
        "content": post_data["html_content"],
        "visibility": "3",      # 0: 비공개, 3: 공개
        "category": post_data["category_id"],
        "tag": ",".join(post_data["tags"]),
        "acceptComment": "1",
    }
    async with httpx.AsyncClient() as client:
        response = await client.post(url, data=params)
        return response.json()
```

### 9.1 게시 스케줄

| 시간 | 작업 |

| 매일 07:00~08:00 | 콘텐츠 생성, 이미지 생성, 코드 검증 (병렬 실행) |
| 매일 08:30 | SEO 최적화 및 품질 검수 |
| 매일 09:00 | 게시 (하루 1~2 포스트) |
| 매주 월요일 09:00 | 주간 성과 리포트 생성 및 발송 |
| 매월 1일 09:00 | 월간 피드백 반영 및 설정값 업데이트 |

### 9.2 운영 환경

```
개발 환경:  Local Docker Compose
스테이징:   AWS EC2 t3.medium + RDS PostgreSQL
프로덕션:   AWS EC2 t3.large 또는 ECS Fargate (Auto Scaling)
```

- **API 호출 비용 임계값 초과:** 비용 경보 알림
